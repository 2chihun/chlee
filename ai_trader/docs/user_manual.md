## 1. 시스템 개요

### 1.1 AI Trader란?

AI Trader는 한국투자증권 API를 활용한 **AI 기반 국내 주식 단타 매매 시스템**입니다.  
기술적 지표 분석과 규칙 기반 전략을 통해 자동으로 매매 신호를 생성하고 실행합니다.

### 1.2 주요 기능

```
┌────────────────────────────────────────────────────┐
│                    AI Trader                        │
│                                                    │
│  ┌──────────┐  ┌──────────┐  ┌──────────────────┐  │
│  │ 데이터   │→ │ 분석     │→ │ 매매 실행        │  │
│  │ 수집     │  │ & 전략   │  │ & 리스크 관리    │  │
│  └──────────┘  └──────────┘  └──────────────────┘  │
│                                                    │
│  ┌──────────┐  ┌──────────┐  ┌──────────────────┐  │
│  │ 백테스트 │  │ 통계     │  │ 대시보드         │  │
│  │ & 시뮬   │  │ & 리포트 │  │ (Streamlit)      │  │
│  └──────────┘  └──────────┘  └──────────────────┘  │
└────────────────────────────────────────────────────┘
```

- **실시간 데이터 수집**: 분봉, 일봉, 실시간 틱 데이터
- **듀얼 전략**: 5분봉 스캘핑 + 인트라데이 스윙
- **자동 매매**: 신호 생성 → 리스크 체크 → 주문 실행
- **백테스트**: 과거 데이터 기반 전략 검증
- **몬테카를로 시뮬레이션**: 수익 확률 분석
- **웹 대시보드**: 실시간 모니터링 및 제어

```
    │       │       ├── 스캘핑 전략
```

## 3. 초기 설정

### 3.1 환경 변수 설정

`.env.example` 파일을 복사하여 `.env` 파일을 생성합니다.

```bash
copy .env.example .env
```

`.env` 파일을 편집합니다:

```ini
# ── 한국투자증권 API ──
KIS_APP_KEY=발급받은_앱키
KIS_APP_SECRET=발급받은_시크릿키
KIS_ACCOUNT_NO=계좌번호-01    # 예: 12345678-01

# ── 거래 모드 ──
TRADING_MODE=paper            # paper=모의투자, live=실전투자

# ── 데이터베이스 ──
DB_USE_SQLITE=true            # SQLite 사용 (기본값)
# PostgreSQL 사용 시:
# DB_USE_SQLITE=false
# DB_HOST=localhost
# DB_PORT=5432
# DB_NAME=ai_trader
# DB_USER=postgres
# DB_PASSWORD=your_password

# ── 리스크 관리 ──
MAX_POSITION_SIZE=2000000     # 종목당 최대 투자금 (원)
MAX_DAILY_LOSS=-100000        # 일일 최대 손실 한도 (원)
MAX_POSITIONS=5               # 최대 동시 보유 종목 수
STOP_LOSS_PCT=-1.0            # 손절 기준 (%)
TAKE_PROFIT_PCT=2.0           # 익절 기준 (%)
```

> ⚠️ **중요**: `.env` 파일은 절대 Git에 커밋하지 마세요!

### 3.2 모의투자 설정 (필수!)

**처음 사용할 때는 반드시 모의투자(paper) 모드로 시작하세요.**

1. 한국투자증권 홈페이지에서 모의투자 계좌 신청
2. 모의투자용 앱키/시크릿키 별도 발급
3. `TRADING_MODE=paper` 설정

### 3.3 감시 종목 설정

`config/settings.py`의 `StrategyConfig`에서 감시 종목을 설정합니다:

```python
watchlist = ["005930", "000660", "035720"]  # 삼성전자, SK하이닉스, 카카오
```

혹은 `.env`에 추가:
```ini
WATCHLIST=005930,000660,035720,035420,051910
```

## 4. 시스템 실행

### 4.1 실행 모드

AI Trader는 세 가지 모드로 실행할 수 있습니다:

| 모드 | 설명 | 명령어 |
|------|------|--------|
| `both` | 봇 + API 서버 통합 실행 (기본) | `python main.py` |
| `bot` | 트레이딩 봇만 실행 | `python main.py --mode bot` |
| `server` | API 서버만 실행 | `python main.py --mode server` |

### 4.2 기본 실행 (봇 + 서버)

```bash
# 가상환경 활성화
.venv\Scripts\activate

# 실행 (모의투자 모드)
python main.py --paper

# 또는 환경 변수 설정 후 실행
python main.py
```

### 4.3 대시보드 실행

별도 터미널에서:

```bash
.venv\Scripts\activate
streamlit run dashboard/streamlit_app.py
```

브라우저에서 `http://localhost:8501` 접속

### 4.4 일일 운용 스케줄


## 6. 전략 설명

### 6.1 스캘핑 전략 (ScalpingStrategy)

**대상**: 5분봉 데이터  
**보유 시간**: 수 분 ~ 수십 분  
**목표 수익**: 0.5% ~ 1.5%  

**매수 조건** (복합 신호):
```
┌─────────────────────────────────────┐
│         스캘핑 매수 조건            │
│                                     │
│  1. RSI < 35 (과매도)          (+1) │
│  2. 종가 < 볼린저 하단밴드    (+1) │
│  3. MACD 골든크로스            (+1) │
│  4. 거래량 > 평균 1.5배       (+1) │
│  5. 종가 < VWAP               (+1) │
│                                     │
│  → 3개 이상 충족 시 매수 신호      │
│  → 신뢰도 = 충족 수 / 5           │
└─────────────────────────────────────┘
```

**매도 조건**:
```
┌─────────────────────────────────────┐
│         스캘핑 매도 조건            │
│                                     │
│  1. RSI > 70 (과매수)          (+1) │
│  2. 종가 > 볼린저 상단밴드    (+1) │
│  3. MACD 데드크로스            (+1) │
│                                     │
│  → 2개 이상 충족 시 매도 신호      │
└─────────────────────────────────────┘
```

**리스크 관리**:
- 손절: ATR × 1.5 (동적)
- 익절: ATR × 2.5 (동적)

### 6.2 스윙 전략 (SwingStrategy)

**대상**: 5분봉 데이터  
**보유 시간**: 30분 ~ 하루  
**목표 수익**: 1.0% ~ 3.0%  

**매수 조건** (3가지 시나리오):

```
┌─────────────────────────────────────┐
│       스윙 전략 매수 시나리오       │
│                                     │
│  A. 눌림목 매수                     │
│     - EMA 정배열 (5 > 20 > 60)     │
│     - RSI 40~60 (중립 구간)        │
│     - MFI < 30 (자금 유입 대기)    │
│     → 신뢰도 0.70                  │
│                                     │
│  B. 갭상승 매수                     │
│     - 갭 상승 발생 (> 1%)          │
│     - 거래량 > 평균 2배            │
│     - RSI < 65                     │
│     → 신뢰도 0.65                  │
│                                     │
│  C. 갭하락 반전 매수                │
│     - 갭 하락 후 3봉 이상 반등     │
│     - 거래량 > 평균 1.5배          │
│     → 신뢰도 0.60                  │
└─────────────────────────────────────┘
```

**매도 조건**:
- RSI > 75 + EMA 역배열
- 트레일링 스탑 작동
- 지지선 하향 돌파

## 7. 백테스트 가이드

### 7.1 기본 백테스트

대시보드 `🔬 백테스트` → `기본 백테스트` 탭:

1. 전략 선택 (스캘핑/스윙)
2. 종목코드 입력 (예: 005930)
3. 초기 자본금 설정
4. "백테스트 실행" 클릭

**결과 지표**:
| 지표 | 설명 |
|------|------|
| 총 수익률 | 전체 기간 누적 수익률 |
| 샤프 비율 | 위험 대비 수익 (1 이상 양호, 2 이상 우수) |
| 최대 낙폭 | 최고점 대비 최대 하락률 |
| 승률 | 이익 거래 비율 |
| Profit Factor | 총이익/총손실 (1.5 이상 양호) |

### 7.2 파라미터 최적화

최적의 전략 파라미터를 자동으로 탐색합니다:

1. `🔬 백테스트` → `파라미터 최적화` 탭
2. 전략, 종목 선택
3. "최적화 실행" 클릭
4. 상위 10개 파라미터 조합 결과 확인

### 7.3 몬테카를로 시뮬레이션

`🎲 시뮬레이션` 메뉴:

1. 전략, 종목 선택
2. 시뮬레이션 횟수 설정 (기본: 1000)
3. 실행

**결과 해석**:
```
┌─────────────────────────────────────┐
│     몬테카를로 결과 해석            │
│                                     │
│  수익 확률 > 60%  → 양호           │
│  수익 확률 > 70%  → 우수           │
│  5% 퍼센타일 > 원금 → 매우 안전    │
│                                     │
│  평균 MDD < 10% → 낮은 위험        │
│  최악 MDD < 20% → 수용 가능        │
└─────────────────────────────────────┘
```

### 7.4 스트레스 테스트

극단적 시장 상황에서의 전략 성과를 확인:

| 시나리오 | 설명 |
|---------|------|
| 폭락장 | 수익률 ×0.7, 변동성 ×2.0 |
| 폭등장 | 수익률 ×1.3, 변동성 ×1.5 |
| 고변동성 | 변동성 ×2.5 |
| 저변동성 | 변동성 ×0.3 |
| 저거래량 | 거래량 ×0.3 |

## 8. 리스크 관리

### 8.1 자동 리스크 체크

모든 매매 신호는 아래 검증을 통과해야 합니다:

```
신호 생성
   │
   ▼
일일 손실 한도 체크  ───(초과)──→ 거부
   │ (통과)
   ▼
최대 포지션 수 체크  ───(초과)──→ 거부
   │ (통과)
   ▼
중복 포지션 체크     ───(있음)──→ 거부
   │ (없음)
   ▼
포지션 크기 산정     ───(부족)──→ 거부
   │ (충분)
   ▼
신뢰도 필터 체크     ───(미달)──→ 거부
   │ (통과)
   ▼
주문 실행 ✅
```

### 8.2 리스크 파라미터

| 파라미터 | 기본값 | 설명 |
|---------|--------|------|
| 종목당 최대 투자금 | 200만원 | 한 종목에 투자하는 최대 금액 |
| 일일 최대 손실 | -10만원 | 초과 시 당일 매매 중단 |
| 최대 동시 포지션 | 5종목 | 동시에 보유 가능한 종목 수 |
| 손절선 | -1.0% | ATR 기반 동적 조절 |
| 익절선 | +2.0% | ATR 기반 동적 조절 |
| 최소 신뢰도 | 0.6 | 60% 미만 신호 무시 |

### 8.3 리스크 관리 권장 사항

1. **처음 2주**: 모의투자로 전략 검증
2. **자본금 대비 1% 규칙**: 한 거래당 최대 손실을 자본금의 1% 이내로
3. **일일 손실 한도**: 자본금의 1~2%로 설정
4. **포지션 분산**: 3~5개 종목에 분산 투자
5. **백테스트 확인**: 실전 전 반드시 백테스트 실행

## 9. FAQ

### Q1. API 키는 어디서 발급받나요?

한국투자증권 OpenAPI 포털 (https://apiportal.koreainvestment.com)에서 발급받을 수 있습니다.
모의투자용과 실전투자용 키가 별도로 필요합니다.

### Q2. 모의투자와 실전투자의 차이는?

| 항목 | 모의투자 | 실전투자 |
|------|---------|---------|
| API URL | openapivts.koreainvestment.com | openapi.koreainvestment.com |
| 자금 | 가상 자금 | 실제 자금 |
| 체결 | 시뮬레이션 | 실제 체결 |
| 수수료/세금 | 가상 | 실제 부과 |

### Q3. 시스템이 손실을 낸다면?

- 일일 손실 한도에 도달하면 자동으로 매매를 중단합니다
- 백테스트 결과가 좋아도 실전에서 다를 수 있습니다
- 반드시 모의투자로 충분히 검증 후 실전에 임하세요
- 리스크 파라미터를 보수적으로 설정하세요

### Q4. 서버가 꺼지면 보유 포지션은?

- 보유 포지션 정보는 DB에 저장됩니다
- 서버 재시작 후 포지션 상태를 확인할 수 있습니다
- 필요 시 한국투자증권 MTS/HTS에서 직접 매도 가능합니다

### Q5. 데이터베이스 변경 방법은?

`.env` 파일에서:
```ini
# SQLite → PostgreSQL 전환
DB_USE_SQLITE=false
DB_HOST=localhost
DB_PORT=5432
DB_NAME=ai_trader
DB_USER=postgres
DB_PASSWORD=your_password
```

### Q6. 새로운 전략을 추가하려면?

1. `strategies/` 디렉토리에 새 파일 생성
2. `BaseStrategy`를 상속하여 구현
3. `analyze()`, `generate_signal()` 메서드 구현
4. `main.py`에서 전략 등록

```python
# strategies/my_strategy.py
from strategies.base import BaseStrategy, Signal, SignalType

class MyStrategy(BaseStrategy):
    @property
    def name(self) -> str:
        return "my_strategy"

    def analyze(self, df):
        # 분석 로직
        pass

    def generate_signal(self, df, stock_code):
        self.analyze(df)
        # 신호 생성 로직
        return Signal(signal_type=SignalType.HOLD, ...)
```
